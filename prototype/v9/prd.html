1. Vision & Problem

Needle helps you "move the needle" — it's a macOS desktop app for software engineers with ADHD that acts as an intelligent second brain. It absorbs anything thrown at it, classifies it automatically, and surfaces the right things at the right time.

Core Insight: People with ADHD generate high volumes of mental input but organizing that input is itself cognitively expensive. Existing tools (Notion, Todoist, Jira) require users to decide where things go. That friction causes avoidance. Needle removes that friction entirely.

A typical day with Needle:

  1. Morning: Open app, dump your day plan (tasks, meetings, ideas) in any format
  2. Work: Focus on one task at a time. Needle tracks time and context
  3. Transition: When it's time to switch, Needle grabs attention → helps brain dump → moves you forward
  4. End of day: Review what you accomplished (celebration, not guilt)

2. Three Pillars

📋

4. Template System

Templates control layout, sections, and visibility — not data. The same data can be displayed in completely different ways.

{ "id": "editorial", "name": "Editorial Journal", "layout": "single-column" | "timeline" | "blocks" | "kanban" | "minimal", "sections": [ { "id": "now", "label": "Now", "filter": "in_progress", "visible": true }, { "id": "coming_up", "label": "Coming Up", "filter": "pending", "visible": true }, { "id": "other", "label": "Other", "filter": "section:other", "visible": true }, { "id": "done", "label": "Done", "filter": "completed", "visible": false } ], "calendar_integration": "woven" | "separate" | "hidden", "show_progress": true, "show_time_markers": true, "show_now_indicator": true, "show_quick_add": true, "greeting": true }
Key Principle: Templates are like "views" on the same database. Users can switch between them freely, or the app can auto-select based on time of day (morning → Editorial, deep work → Minimal, evening → Kanban).

  1. Parse: Regex scan for @mentions, JIRA-XXX, URLs, time expressions
  2. Resolve: Look up Jira URLs, Slack user IDs, URL metadata
  3. Store: Raw text + links array (keeps text editable, links separately)
  4. Render: Replace patterns with interactive chips/links at display time

6. ADHD Design Research

Why ADHD Users Abandon Todo Apps

ReasonOur Solution
Task initiation gapAI breaks tasks into micro-steps. "What's the first thing to do?" prompt.
Working memory / "object permanence"Widget, notifications, macOS menu bar presence. Impossible to forget.
Time blindnessVisual timers (pie-chart style), "now" indicators, timeline view.
Novelty dopamine crash (3-week collapse)Template switching, varied interruptions, "Welcome Back" flow.
Overwhelm from long listsMax 7 items visible. Auto-collapse. "Focus Mode" = 1 task.
Shame spiralsNO red overdue indicators. No punitive language. Easy reset.

The "Interest-Based Nervous System"

ADHD brains are NOT motivated by importance or future rewards. They're powered by novelty, urgency, and immediate gratification. Systems that force neurotypical discipline will fail.

The 3-Must-Dos Rule: Never list more than 3 highlighted priorities per day. Long lists trigger task paralysis. Everything else is "bonus" or hidden.

Attention Residue (Dr. Sophie Leroy)

When you switch tasks, your brain continues processing "open loops" from the previous task. This residue persists throughout the subsequent task. It's thicker when the previous task was incomplete. ADHD amplifies this effect.

→ This is why Brain Dump exists. Writing "where I left off" closes the loop and reduces residue.

8. Interruption Design

Escalation Model

Interventions escalate through 5 intensity levels:

LevelTypeExamplesTiming
1AmbientTimer bar, breathing border, color shiftT-10 min
2NudgeDynamic island, toast notification, gentle shakeT-5 min
3TakeoverTorch, frost glass, vignette, split revealT-2 min
4CharacterKitty, owl, clippy, penguin paradeT-1 min
5Full BlockSpotlight command, brain dump promptT-0
Critical Rule: User can ALWAYS dismiss. Never lock them out. Collect the signal (they dismissed at T-2 for the 3rd time today) and learn. Always shameless — it's OK.

Dismissal Options

  • "5 more minutes" — Snooze. Resets to level 1.
  • "Brain Dump & Transition" — Opens brain dump flow, then transitions.
  • "Skip to next" — Jump directly to next task, no dump.
  • "Dismiss" — Cancel this interruption entirely. Optional reason.

10. Anti-Patterns — What NOT To Do

Never do these:
  • ❌ Red for urgency — Use warm terracotta (#C4622D) instead. Red triggers anxiety.
  • ❌ Pure black on pure white — Causes eye strain (halation). Use #1F1A14 on #F6F1EB.
  • ❌ Overdue counts — No "5 tasks overdue!" badges. No guilt.
  • ❌ Auto-playing media — No sounds unless explicitly enabled.
  • ❌ Modal heavy UX — Modals steal context. Use inline/overlay patterns.
  • ❌ Dense information — Max 7 items visible. Progressive disclosure for everything else.
  • ❌ Generic empty states — "Nothing here yet" is lazy. Use warm, encouraging copy.
  • ❌ Punitive gamification — No losing streaks. Streaks freeze, never break.
  • ❌ Complex onboarding — Should work immediately with minimal setup.
  • ❌ Strict time limits — Always allow extend/pause/dismiss.
The Needle Mantra: "A calm, warm room where everything you need is exactly where you'd expect it, nothing screams for attention, and checking something off feels like a tiny celebration."
